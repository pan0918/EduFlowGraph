# 用户画像重构

## 将用户画像分为三个模型：

**学习者模型**不是单纯“存学生能力”，而是描述“学生当前是怎样的学习者”。

**教学策略模型**不是单纯“存学生喜欢什么教学风格”，而是描述“对这个学生，什么教学干预更可能有效”。

**情境模型**也不只是“存学生学习状况”，而是描述“学生此刻处在什么学习场景、任务压力和状态约束中”。

它不是只知道学生是谁，而是知道：

> 这个学生现在处于什么状态，为什么会这样，我应该怎么帮他，以及当前场景下什么帮助最合适。

---
# 一、学习者模型：学生“是什么样的学习者”

把学习者模型理解成 Agent 对学生的“学习内在状态”的理解。

它关注的是学生自身，而不是教学方法。

比如一个学生学不好某个概念，学习者模型要回答的不是：

“他喜欢视频还是文字？”

而是：

“他到底卡在哪里？”

可能是基础概念没建立，可能是迁移能力弱，可能是表达不出来，可能是以为自己懂了但其实只记住了表面定义。

所以学习者模型的核心不是“能力高低”，而是更细的东西：

> 学生的知识结构、认知特点、理解方式、薄弱点、误区、学习习惯和发展潜力。

它像是教育 Agent 的“诊断系统”。

如果没有学习者模型，Agent 就只能泛泛讲课；有了学习者模型，Agent 才能判断学生为什么学不会。

举个概念上的例子：

两个学生都说“我不懂用户画像”。

但背后的学习者状态可能完全不同。

一个学生是不知道定义，属于概念缺失。

另一个学生知道定义，但不知道怎么用于系统设计，属于应用迁移困难。

还有一个学生能说出很多词，但无法组织成框架，属于结构化表达困难。

如果 Agent 只有“学生不懂用户画像”这个粗略判断，就只能重复解释概念。

但如果学习者模型足够细，Agent 就能区分：

这个人需要补定义，还是需要案例，还是需要设计框架，还是需要论文表达训练。

这就是学习者模型的价值。

---

# 二、教学策略模型：应该“怎么教这个学生”

教学策略模型关注的不是学生本身，而是 Agent 的教学行为。

也就是说，学习者模型回答：

> 学生现在是什么状态？

教学策略模型回答：

> 面对这种状态，我应该采取什么教学方式？

这两者一定要区分开。

很多教育系统容易把它们混在一起，比如把“学生喜欢案例教学”直接放进学习者画像里。但严格来说，这更像是教学策略模型的一部分。

因为它不是描述学生会什么，而是在描述：

“什么样的教学方式对这个学生更有效。”

教学策略模型可以理解成 Agent 的“教学决策偏好”。

它不是简单记住“学生喜欢幽默一点”或者“学生喜欢结构化”，而是更进一步：

什么情况下应该讲解？

什么情况下应该追问？

什么情况下应该给提示？

什么情况下应该直接给答案？

什么情况下应该让学生自己生成？

什么情况下应该降低难度？

什么情况下应该提高挑战？

这才是教学策略模型真正有意思的地方。

---

## 教学策略模型不是固定风格，而是动态策略

这一点很重要。

不要把教学策略模型设计成：

> 这个学生喜欢图文并茂，所以永远图文并茂。

这样会变成静态偏好标签。

更好的理解是：

> 对这个学生，在不同学习状态下，哪种教学策略更有效。

比如同一个学生：

刚接触新概念时，需要框架化讲解。

进入理解阶段时，需要案例对比。

准备考试时，需要高频练习。

写论文时，需要学术表达和结构反馈。

遇到挫败时，需要降低认知负荷。

已经掌握后，需要挑战性任务。

所以教学策略模型不是“学生喜欢什么”，而是“Agent 应该怎么调节教学行为”。

它像是教育 Agent 的“处方系统”。

学习者模型负责诊断，教学策略模型负责开处方。

---

# 三、情境模型：学生“此刻处在什么学习环境中”

情境模型是很多教育 Agent 容易忽视的一层，但它其实很关键。

因为教育不是发生在真空中的。

同一个学生、同一个知识点，在不同情境下，Agent 的帮助方式应该完全不同。

比如学生问：

“帮我讲讲用户画像。”

如果他是在刚入门阶段，Agent 应该讲基础概念。

如果他是在写论文，Agent 应该帮他形成理论表达。

如果他明天要汇报，Agent 应该帮他组织成汇报逻辑。

如果他已经很疲惫，Agent 应该压缩信息密度。

如果他正在赶 deadline，Agent 不应该展开太多理论，而应该优先帮他完成任务。

这就是情境模型的意义。

它回答的是：

> 现在这个学习行为发生在什么背景下？

也就是说，情境模型不是单纯记录“学生在哪里学习”“什么时候学习”，而是理解当前学习行为的外部条件。

它包括任务背景、时间压力、学习阶段、情绪状态、资源条件、当前对话目的等。

只要先抓住这个思想：

> 情境模型决定 Agent 此刻应该“以什么优先级”帮助学生。

---

# 这三个模型的关系

我觉得可以用一个很直观的关系来理解：

```text
学习者模型：他现在是什么状态？
教学策略模型：我应该怎么教他？
情境模型：现在这种情况下，什么帮助最合适？
```

或者更像医学里的逻辑：

```text
学习者模型 = 诊断
教学策略模型 = 治疗方案
情境模型 = 当前病情与环境约束
```

比如医生看病，不只是知道你“身体不好”。

医生还要知道：

你具体哪里有问题，这是诊断。

你适合吃药、手术、休息还是观察，这是治疗方案。

你现在是否发烧、是否怀孕、是否要马上上飞机，这是情境约束。

教育 Agent 也是一样。

它不能只知道学生“基础差”。

它要知道学生具体哪里差，适合怎么补，以及当前场景下该不该补。

---

# 一个更完整的理解方式

把这三个模型理解成教育 Agent 的三个视角：

## 学习者模型：看学生

它关注学生的内部学习状态。

它让 Agent 形成对学生的理解。

比如：

这个学生是概念没懂，还是不会应用？

是缺知识，还是缺方法？

是不会表达，还是没有结构？

是短期卡住，还是长期薄弱？

所以学习者模型偏“认知诊断”。

---

## 教学策略模型：看教学

它关注 Agent 该如何介入。

它让 Agent 能选择教学动作。

比如：

是解释，还是提问？

是给例子，还是让学生自己举例？

是鼓励，还是纠错？

是拆步骤，还是提高难度？

是给完整答案，还是只给线索？

所以教学策略模型偏“教学决策”。

---

## 情境模型：看当下

它关注当前学习行为发生的具体场景。

它让 Agent 不至于机械教学。

比如：

学生现在是在探索新知识，还是完成作业？

是在备考，还是做项目？

是有充足时间，还是马上要交？

是想深入理解，还是只想快速解决？

是精力充沛，还是已经焦虑？

所以情境模型偏“场景适配”。

---

# 一个很关键的想法：三者不是并列静态模块，而是循环系统

这三个模型不是各存各的，而是不断互动。

比如 Agent 给学生讲了一个概念。

学生听完后追问：“能不能举个例子？”

这说明学习者模型可能要更新：学生对抽象定义理解不足。

教学策略模型也要更新：这个学生在该类概念上更适合案例驱动。

情境模型也要更新：当前对话进入了概念澄清阶段，而不是任务产出阶段。

然后 Agent 下一轮回答就会改变。

所以这个系统应该是一个循环：

```text
观察学生表现
    ↓
更新学习者模型
    ↓
选择教学策略
    ↓
结合当前情境
    ↓
生成教学行为
    ↓
再次观察学生反应
```

这个循环比“建立一次画像然后一直用”更符合教育过程。

---

# 我觉得最有潜力的表述

概括成一句话：

> 教育 Agent 的用户画像不是对学生静态特征的描述，而是由学习者模型、教学策略模型和情境模型构成的动态学习支持系统。其中，学习者模型用于理解学生，教学策略模型用于决定如何教学，情境模型用于判断当前场景下何种支持最合适。
